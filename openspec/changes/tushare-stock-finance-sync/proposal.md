## Why

我们需要获取A股上市公司的财务指标数据（`fina_indicator`），用于后续的量化分析、基本面选股及财务健康度监控。目前系统已经支持了股票基础信息和每日行情数据的同步，但缺失关键的财务基本面数据。为了构建全面的数据底座，我们必须补全这些财务数据。

## What Changes

- **增量同步功能**：提供按公告日期（`ann_date`）或单只股票（`ts_code`）维度的增量同步接口，供日常定时任务增量拉取最新发布的财报指标。
- **全量同步机制**：提供一次性全市场财务指标同步能力（通过后台任务处理 Tushare 频率限制，拉取全市场历史指标数据，并支持断点续传机制，防止因异常中断）。
- **领域与基础设施层扩展**：引入财务数据对应的领域实体 (Entities)、仓储接口 (Repositories) 以及对接 TuShare 的基础设施网关 (Gateways)。

## Capabilities

### New Capabilities
- `sync-finance-indicator`: 提供拉取、处理、并存储股票财务指标（`fina_indicator`）的能力。支持全量历史异步拉取和基于公告日期的增量更新。

## Impact

- **数据层**：新增财务指标相关的数据库表与 ORM 模型，需要相应的数据库迁移脚本。
- **基础设施层**：新增针对 `fina_indicator` 接口的 Gateway 实现。需复用或扩展原有的令牌桶限流逻辑。
- **应用层**：新增相应的 Command 和 Handler，同时需引入专门针对全量同步任务的状态记录（通过 Redis 或 DB），以便支持断点续传。
